# 下一版交接计划：v1.1「真实学习闭环版」

日期：2026-05-05

## 1. 版本目标

下一版不再追求“把看得到的入口都做真”，而是进一步收窄为“只保留对学习日语有直接作用的真实入口”。首页搜索、文章分享、文章收藏主按钮都暂时移出范围，避免把版本容量花在新闻浏览、传播或半成品收藏上。

核心目标是让用户完成一条最短学习闭环：

```text
读文章 -> 点词/保存词 -> 去单词本沉淀 -> 去复习 -> 看见真实学习进度
```

成功标准：

- 首页不再出现无学习必要的搜索入口。
- 文章页不再出现分享按钮。
- 文章页底部不再保留“收藏文章”主按钮。
- 文章页底部只保留真实学习动作。
- 我的页不再出现假登录、假设置、假连续天数。
- 连续天数和最近 30 天活动格来自真实本地学习状态。
- 复习页进度条按本轮复习进度真实变化。
- 保持现有文章 API 不变。

## 2. 推荐实施范围

### 2.1 首页去掉搜索入口

目标文件：

```text
src/components/home-screen.tsx
```

要求：

- 删除首页顶部“搜索新闻、单词或主题”的搜索框。
- 不新增 `q` 参数。
- 不新增首页搜索逻辑。
- 保留：
  - 品牌区
  - 频道切换
  - 分类筛选
  - 今日复习卡
  - 文章列表
- 首页的核心任务保持为“快速选一篇适合难度的文章开始读”。

不做：

- 不做首页搜索。
- 不做新闻全文搜索。
- 不新增搜索 API。

### 2.2 文章页底部改成学习动作

目标文件：

```text
src/components/article-learning-controls.tsx
```

要求：

- 移除“分享”按钮。
- 移除文章页底部的“收藏文章”主按钮。
- 底部操作栏改为学习闭环相关动作：
  - `去复习`
  - `去单词本`
  - `回首页` 或 `继续阅读` 状态提示
- 打开文章仍然记录已读历史。
- 点击正文词条后，保存单词 / 取消保存单词能力保持不变。
- 单词保存后，文案继续引导用户进入单词本和复习页。

推荐按钮设计：

```text
去复习      -> /review
去单词本    -> /words
回首页      -> /
```

如果为了减少底部按钮数量，也可以保留两个主动作：

```text
去复习      -> /review
去单词本    -> /words
```

并把“回首页”放在文章顶部原有返回入口中，不放入底部操作栏。

不做：

- 不接入 Web Share API。
- 不做复制链接。
- 不生成分享海报。
- 不做文章收藏列表。
- 不把文章收藏只作为统计保留在主路径中。

### 2.3 文章收藏暂时降级

当前 `savedArticleIds` 已存在，但没有收藏文章列表承接。v1.1 不把“收藏文章”作为主学习动作。

处理原则：

- 不删除 `learning-store` 里的文章收藏能力，避免破坏已有状态。
- 不在文章页底部主操作栏展示收藏文章按钮。
- 我的页可以不展示收藏文章数，或将其弱化为历史兼容数据。
- 后续如果要恢复文章收藏，必须同时做：
  - 收藏文章列表
  - 稍后读入口
  - 复读材料库入口

### 2.4 我的页改成本地学习模式

目标文件：

```text
src/components/me-screen.tsx
```

要求：

- 将“未登录 + 登录/注册”改成“本地学习模式”。
- 文案说明当前学习记录保存在本机浏览器。
- 移除不可用入口：
  - 登录 / 注册
  - 复习提醒
  - 默认频道
  - 字体大小
  - 隐私政策
  - 帮助与反馈
- 不保留可点击但无功能的设置项。

推荐展示：

- 本地学习模式卡片。
- 已读文章数。
- 保存单词数。
- 今日待复习数。
- 今日已复习数。
- 当前连续学习天数。
- 最近 30 天学习活动格。

### 2.5 连续天数和活动格真实化

目标文件：

```text
src/lib/learning-store.ts
src/components/me-screen.tsx
src/components/review-screen.tsx
```

要求：

- 不改变 `qingdu-learning-state-v1` schema。
- 在 `learning-store` 增加派生计算函数。
- 活动日期来源：
  - `readHistory.readAt`
  - `savedWords.savedAtIso`
  - `reviewRecords.reviewedAt`
- 当前连续学习天数从今天往前计算。
- 如果今天没有活动但昨天有活动，连续天数可以从昨天开始计算，避免用户当天还没学习时直接归零。
- 最近 30 天活动格按日期生成。
- 活动强度建议：
  - 0 次：`empty`
  - 1 次：`mid`
  - 2 次及以上：`dark`

建议新增函数：

```ts
getTodayReviewCount(state: LearningState, now?: Date): number
getLearningActivity(state: LearningState): Map<string, number>
getCurrentStreak(state: LearningState, now?: Date): number
getRecentActivityCells(state: LearningState, now?: Date): ("empty" | "mid" | "dark")[]
```

### 2.6 复习页进度真实化

目标文件：

```text
src/components/review-screen.tsx
```

要求：

- 本轮复习开始时固定一个 `initialQueueLength`。
- 显示真实进度，例如：
  - `已完成 0 / 5`
  - `1 / 5`
- 每完成一个词，进度条按完成比例更新。
- 当前队列变空时显示本轮完成态。
- 今日待复习数继续来自真实学习状态。
- “连续 7 天”改为真实连续天数。

不做：

- 不引入复杂 SRS 算法。
- 不改变当前 `again / hard / good / easy` 间隔。

## 3. 接口与兼容性

必须保持不变：

```text
GET /api/articles?channel=easy|original&category=...
GET /api/articles/[id]
```

必须保持不变：

```text
localStorage key: qingdu-learning-state-v1
```

本阶段不做 schema migration。所有新增数据都应通过现有字段派生计算。

## 4. 测试计划

### 4.1 自动测试

更新：

```text
scripts/learning-state-smoke.mjs
```

至少覆盖：

- 今日复习数计算。
- 当前连续学习天数计算。
- 今天无活动但昨天有活动时，连续天数仍能合理展示。
- 最近 30 天活动格长度固定为 30。
- 活动强度能输出 `empty` / `mid` / `dark`。
- 原有保存单词、取消保存单词、复习评分测试仍通过。

完成后运行：

```powershell
npm.cmd run test:learning
npm.cmd run lint
npm.cmd run build
```

如时间允许，再运行：

```powershell
npm.cmd run test
```

### 4.2 浏览器验收

在 `http://localhost:3100` 手动验收：

1. 首页不再出现搜索框。
2. 首页频道切换、分类筛选、文章列表仍正常。
3. 文章页底部不再出现分享按钮。
4. 文章页底部不再出现收藏文章主按钮。
5. 文章页底部只保留真实学习动作，且点击后能进入对应页面。
6. 文章词条弹层仍能保存 / 取消保存单词。
7. 单词保存后 `/words` 可见。
8. `/review` 每评分一个词，进度数字和进度条同步变化。
9. `/me` 不再有假登录、假设置入口。
10. 阅读文章、保存单词、完成复习后，我的页统计和活动格变化。

## 5. 明确不做

本版本不做：

- 首页搜索。
- 文章分享。
- 文章收藏列表。
- 账号登录。
- 多设备同步。
- 服务端学习状态 API。
- 推送通知。
- 字体设置功能。
- 第三方分享 SDK。
- 搜索后端或全文索引。
- 新内容源。
