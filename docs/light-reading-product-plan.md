# 兴趣驱动的日语轻阅读改造计划

## 1. 改造目标

当前项目已经具备新闻抓取、文章解析、分词、词典、学习状态和阅读页等基础能力。

本次改造不是重构，而是把现有的新闻阅读项目，最小成本迁移为：

> 面向中国日语学习者的兴趣驱动日语轻阅读 PWA。

核心体验：

```text
每天 3-5 分钟，读一点真正感兴趣的日语。
```

最终方向：

```text
新闻阅读
→ 轻阅读
→ 兴趣驱动
→ 青空文库短段
→ 分段长阅读
→ 生词闭环
```

---

## 2. 为什么要改

### 2.1 当前新闻路线的问题

NHK 等新闻全文存在版权与合规风险，不适合作为长期内容核心。

新闻源还会带来：

- 外部链接在中国大陆访问不稳定
- 新闻内容偏严肃，二次元和兴趣型学习用户不一定愿意打开
- 内容更新依赖外部站点结构
- 学习体验容易变成资讯浏览
- 独立开发维护成本较高

### 2.2 新路线的优势

兴趣驱动轻阅读更贴近中国日语学习用户，尤其是二次元、游戏、旅行、日本生活兴趣用户。

用户打开理由从：

```text
我要看日本新闻
```

变成：

```text
我想看懂推し文化
我想看懂动画表达
我想看懂游戏公告
我想读懂真实日语
```

技术上也更稳定：

- 内容可控
- 不依赖新闻全文抓取
- 可以使用原创学习文章
- 可以接入青空文库短段
- 现有 tokenizer、dictionary、learning state 都可以继续复用

---

## 3. 新产品定位

旧定位：

```text
Japanese News Reading PWA
```

新定位：

```text
Interest-driven Japanese Light Reading
兴趣驱动的日语轻阅读
```

产品一句话：

```text
每天 3-5 分钟，读一点你真正感兴趣的日语。
```

产品不是：

- 新闻客户端
- 完整电子书库
- 动漫资料站
- Anki 替代品

产品是：

```text
轻量阅读 + 点词查词 + 生词收藏 + 分段成长
```

---

## 4. 保留现有技术资产

现有能力继续复用：

- Article 展示结构
- 阅读页
- 文章解析逻辑
- tokenizer 分词
- dictionary 查询
- selection 逻辑
- learning state
- PWA 基础

本次改造重点不是重写底层，而是改变：

```text
内容类型
首页心智
阅读页文案
文章组织方式
```

---

## 5. 内容模型改造

### 5.1 从 NewsArticle 到 ReadingArticle

不建议大规模重构数据结构。

当前文章模型可以先继续使用，只在语义上从新闻文章改为阅读文章。

建议逐步抽象为：

```ts
type ReadingContentType =
  | "interest_japanese"
  | "life_japanese"
  | "culture_japanese"
  | "aozora_segment"
  | "news_learning";
```

### 5.2 新增轻量字段

```ts
type ReadingArticle = {
  id: string;
  title: string;
  body: string;
  summary?: string;
  source?: string;
  sourceUrl?: string;

  contentType: ReadingContentType;
  theme?: string;
  level?: "N3" | "N2" | "N1";
  estimatedMinutes?: number;
  tags?: string[];

  seriesId?: string;
  segmentIndex?: number;
  segmentTotal?: number;
  prevArticleId?: string;
  nextArticleId?: string;
};
```

### 5.3 设计原则

青空文库、兴趣日语、生活日语、新闻学习摘要都先统一走 Article。

不要第一阶段新建：

```text
Book
Chapter
Bookshelf
ReaderEngine
```

长阅读也先拆成多个 Article，通过 seriesId 串联。

---

## 6. 内容策略

### 6.1 主力内容：兴趣日语

面向二次元、游戏、推し活、J-POP、VTuber、轻小说兴趣用户。

内容必须原创或学习化改写，不直接搬运动画、漫画、歌词、游戏剧情或轻小说原文。

示例：

```text
「推し」ってどういう意味？
アニメ感想でよく見る「尊い」
ゲームのお知らせに出る「実装」
「復刻」と「再販」は何が違う？
配信でよく聞く「初見です」
グッズ販売でよく見る「予約受付中」
```

### 6.2 辅助内容：日本生活日语

适合旅行、留学、日本生活兴趣用户。

示例：

```text
コンビニでよく見る「温めますか」
駅で聞こえる「まもなく到着します」
飲食店で使う「店内で」「持ち帰りで」
ホテル予約で見る「禁煙」「素泊まり」
```

### 6.3 成长内容：青空文库短段

青空文库不是现代二次元内容库，但有很多二次元文化原型和文豪作品。

优先选择有兴趣入口、氛围强、适合短段的作品：

```text
蜘蛛の糸
羅生門
山月記
走れメロス
銀河鉄道の夜
吾輩は猫である
```

包装方式不要叫“近代文学阅读”，而是：

```text
经典短读
幻想系
黑暗系
中二系
文豪短读
```

### 6.4 降级内容：新闻学习

新闻可以保留，但不再是主身份。

新闻只做：

- 学习摘要
- 关键词
- 背景知识
- 来源标注
- 参考链接

不做新闻全文站内展示。

---

## 7. 首页改造

### 7.1 一级导航

保持极简：

```text
今日
生词
我的
```

不要做复杂频道导航：

```text
动漫
游戏
新闻
文学
旅行
IT
文化
```

内容可以多样，入口必须轻。

### 7.2 今日页结构

把原新闻列表页改成：

```text
今日の一篇

「推し」ってどういう意味？
N3-N2 · 3分 · 兴趣日语

[开始阅读]

继续阅读
最新阅读
```

保留现有列表，但从“新闻列表”改成“最新阅读”。

### 7.3 首页设计原则

- 一屏只突出一篇
- 不制造选择压力
- 不强调来源复杂度
- 显示阅读时间
- 显示难度
- 显示内容类型

---

## 8. 阅读页改造

阅读页从新闻详情页改成学习阅读页。

核心结构：

```text
标题
难度
预计时间
内容类型
正文
点词
收藏
参考来源
```

文案替换：

| 旧文案 | 新文案 |
|---|---|
| 新闻 | 阅读 |
| 新闻分类 | 阅读主题 |
| 今日新闻 | 今日の一篇 |
| 查看原文 | 参考来源 |
| 原文链接 | 来源页面 |

点词弹窗保持轻量，默认只展示：

```text
单词
假名
中文释义
词性
例句
收藏按钮
```

AI 解释默认折叠，不要阻断阅读。

---

## 9. 分段长阅读方案

不要重做电子书阅读器。

长文拆成多个 Article：

```text
rashomon-001
rashomon-002
rashomon-003
```

通过字段串联：

```ts
seriesId: "rashomon"
segmentIndex: 1
segmentTotal: 8
nextArticleId: "rashomon-002"
```

页面展示：

```text
羅生門
第 1 段 / 全 8 段 · 4分

[上一段] [下一段]
```

这样可以实现长阅读成长线，但不用重构为书库系统。

---

## 10. 生词系统

第一阶段只做最近收藏。

展示字段：

```text
单词
假名
中文释义
来源文章
出现句子
```

暂不做：

```text
复杂 SRS
遗忘曲线
熟练度
多级分类
排行榜
```

生词系统的第一目标不是替代 Anki，而是让阅读形成闭环。

## 11. 落地路线

### Phase 1：UI 文案迁移

目标：新闻阅读改成轻阅读心智。

改动：

- 新闻 → 阅读
- 新闻分类 → 阅读主题
- 今日新闻 → 今日の一篇
- 查看原文 → 参考来源
- 新闻列表 → 最新阅读

### Phase 2：新增 contentType

目标：让不同内容源统一走 Article。

新增字段：

```text
contentType
theme
level
estimatedMinutes
```

暂时不重构数据库核心。

### Phase 3：首页轻阅读化

目标：让用户打开就知道今天读什么。

改动：

- 顶部新增今日一篇卡片
- 保留现有列表
- 最新列表从新闻流改成阅读流

推荐逻辑先简单处理：

```text
固定推荐
或从文章池随机选择
```

### Phase 4：接入 10 篇种子内容

不要先自动化。

先手动准备 10 篇，用来验证方向。

建议：

兴趣日语 5 篇：

```text
推し
尊い
実装
復刻
配信
```

日本生活 3 篇：

```text
便利店
电车
菜单
```

青空文库 2 篇：

```text
羅生門
蜘蛛の糸
```

### Phase 5：分段长阅读

目标：让青空文库成为成长线。

新增字段：

```text
seriesId
segmentIndex
segmentTotal
prevArticleId
nextArticleId
```

阅读页底部增加上一段/下一段。

### Phase 6：最近收藏

复用 learning state。

生词页先只展示最近收藏，不做复杂复习系统。

## 12. 暂时不要做

- 不做完整新闻全文抓取
- 不做大型青空文库书库
- 不做多频道首页
- 不做动漫资料站
- 不抓取动漫、漫画、歌词、游戏剧情原文
- 不做复杂 AI 老师
- 不做复杂 SRS
- 不做社交社区
- 不重构成大型学习平台

---

## 13. 成功指标

第一阶段优先看：

```text
今日一篇点击率
文章读完率
点词次数
收藏单词数
次日再次打开率
```

第二阶段再看：

```text
分段长阅读完成率
下一段点击率
最近收藏查看率
继续阅读点击率
```

不优先看：

```text
文章总数量
频道数量
新闻来源数量
浏览文章数量
```

这些数字容易让产品变重。

---

## 14. 内容与合规原则

### 兴趣日语

- 做原创学习文章
- 不搬运动画、漫画、歌词、游戏剧情、轻小说正文
- 可以讲词语、表达、场景和文化背景

### 青空文库

- 保留作品名、作者、来源信息
- 优先短段展示
- 不把首页做成书库
- 作为分段长阅读和文豪兴趣入口

### 新闻学习

- 不站内展示新闻全文
- 只做学习摘要、关键词、背景知识
- 来源链接作为参考
- 链接打不开不影响站内阅读体验

---

## 15. 最终体验

用户打开 App：

```text
今日の一篇

「尊い」为什么经常出现在动画感想里？
N3-N2 · 3分 · 兴趣日语
```

用户进入阅读页：

- 读短文
- 点词查义
- 收藏生词
- 读完获得完成感

第二天：

```text
继续阅读
羅生門 第 2 段
```

产品既保留轻量入口，也有长期成长线。

---

## 16. 关键原则

你真正要保护的不是新闻源，而是：

```text
阅读体验
开始成本
点词流畅度
完成感
```

内容可以慢慢换。

但用户是否愿意每天打开读 3 分钟，才是产品真正的核心。
