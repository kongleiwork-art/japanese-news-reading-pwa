# Frontend QA Checklist

## 适用范围

这份清单用于移动端优先的 React / Next.js / PWA 项目，面向三类协作者共同使用：

- 开发：自测交互、状态、路由和实现完整性
- 设计：验收视觉一致性、信息层级和设计还原度
- 测试：按页面与状态矩阵做回归验证

## 使用方式

- 不要只验单个页面默认态，要验“页面 + 状态”组合
- 每次发版前至少完整跑一轮主链路
- 对于未实现能力，不允许保留“看起来能点但没有行为”的按钮

## 一、主链路验收

### 首页

- 默认频道加载正常
- 频道切换正常
- 分类筛选正常
- 搜索框占位文案正确
- 每一张新闻卡都能进入真实详情页
- 复习入口按钮可见且可点击

### 新闻详情页

- 返回首页正常
- 顶部标题不溢出
- 文章标签、来源、发布时间、阅读时长显示正确
- 正文排版稳定，无异常换行和遮挡
- 可点击单词能进入词条详情状态
- 已保存单词区与正文点词行为一致
- 底部操作栏不遮挡正文

### 单词本

- 分组默认展开/折叠状态符合预期
- 点击单词可进入词条详情
- 单词读音、释义、等级、保存时间显示正确
- 删除按钮视觉位置正确
- 页面顶部入口按钮有真实行为或锚点

### 复习页

- 正面卡片可翻到背面
- 背面卡片可回到正面
- 查看详情入口正常
- 三个掌握度按钮文案清晰、位置正确
- 来源信息不被遮挡
- 翻面过程中无重影、叠层、内容重叠

### 我的页

- 顶部设置入口正常
- 登录区块完整
- 统计卡数字和标签对齐
- 学习设置与应用设置列表层级清楚
- 列表项箭头、间距、分割线统一

## 二、状态矩阵验收

至少覆盖以下 URL / 状态：

- `/`
- `/?channel=original`
- `/?category=经济`
- `/article/ai-lead`
- `/article/ai-lead?word=technique`
- `/words`
- `/words?word=ai`
- `/review`
- `/review?side=back`
- `/review?side=back&word=growth`
- `/me`

验收标准：

- 页面返回 `200`
- 当前选中态与 URL 一致
- 关闭或返回后回到正确基页
- 刷新页面后状态仍可恢复

## 三、按钮与跳转验收

- 每个主按钮必须有真实去向
- 每个图标按钮必须有明确语义
- 每个底部导航项必须能跳到对应页
- 每个卡片点击区域要稳定命中
- 不允许出现“假按钮”

假按钮定义：

- 看起来可点击
- 但没有跳转、状态变化或明确提示

处理原则：

- 要么补真实行为
- 要么降级为非交互视觉

## 四、视觉一致性验收

### 容器与布局

- 所有页面都保持统一移动端画布宽度
- fixed / sticky 元素与主容器对齐
- 无桌面宽屏化拉伸
- 无父级 `overflow` 裁切弹层

### 字体与文案

- 中文 UI 字体自然，不像日文字体硬替换
- 中文 UI 与日文内容分层明确
- 无乱码、编码污染、问号占位异常
- 主 CTA 文案完整可见

### 组件系统

- 圆角统一
- 阴影统一
- 边框浓度统一
- 按钮高度和对齐统一
- 标签 chips 高度统一

## 五、交互稳定性验收

- 弹窗可稳定打开与关闭
- 弹窗不会出现在可视区外
- 弹窗不会被遮罩或父容器裁切
- 翻面状态稳定，不依赖脆弱瞬时状态
- in-app browser 下与普通浏览器下行为尽量一致

## 六、数据一致性验收

- 首页文章列表与详情页数据一一对应
- 分类枚举与显示文案一致
- 频道值与标签文案一致
- 词条数据、正文点词、单词本数据来自同一真源
- 不允许页面本地散写旧枚举或临时字符串

## 七、开发自测要求

- 跑 `lint`
- 跑 `TypeScript` 检查
- 本地启动后按状态矩阵至少走一轮
- 每次修复交互问题后必须回归相关页面，不只看当前页

## 八、设计验收要求

- 不只看“像不像稿子”，还要看：
  - 操作是否真实
  - 状态是否闭环
  - 页面节奏是否统一
  - 字体、按钮、卡片是否达到上线产品标准

## 九、测试回归要求

- 每次改数据层，要回归所有依赖该数据的页面
- 每次改弹窗，要回归文章页、单词本、复习页
- 每次改底部导航，要回归 4 个主页面
- 每次改容器布局，要回归详情页和 fixed 元素

## 十、发版前阻塞项

有以下任一问题，不允许进入“可上线”状态：

- 首页卡片存在死链
- 词条详情无法稳定打开
- 复习翻面异常
- 主按钮文字不可见
- 存在明显乱码
- 页面宽度或弹层定位明显异常
- 当前选中态与 URL 不一致
