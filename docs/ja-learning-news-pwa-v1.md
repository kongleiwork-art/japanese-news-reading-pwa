# 日语新闻学习 App V1 设计方案

## 1. 产品定位

这是一款面向中文用户的轻量级日语学习 PWA，核心路径不是传统课程，而是“用最新日语新闻学日语”。

V1 聚焦两个频道：

- 简单频道：以 `NHK NEWS WEB EASY` 为核心，提供适合初中级学习者的简明日语新闻。
- 原文频道：提供更接近真实母语环境的日语新闻流，V1 直接使用 `NHK NEWS WEB` 正常新闻流，和简单频道形成同源双难度结构。

产品目标不是做“大而全”的日语学习平台，而是做“每天打开 10 分钟就能学到真实日语”的新闻型学习工具。

## 2. 目标用户

- 中文母语、日语 N5-N3 阶段学习者
- 想通过真实内容提高阅读、词汇和语感的人群
- 不想做重度刷题，更偏向碎片化学习的人

## 3. V1 产品目标

- 每天稳定提供最新新闻内容
- 让用户能在 30 秒内选到适合自己难度的文章
- 让一篇新闻具备“阅读 + 生词 + 句子理解 + 收藏”的最小闭环
- 以轻量 PWA 形式先验证留存，不先做重客户端

不在 V1 范围内：

- 系统化课程
- AI 口语陪练
- 社区
- 复杂的等级测试
- 多语言 UI

## 4. 为什么 V1 选 PWA

- 开发和迭代成本最低，适合快速验证内容型产品
- 中文用户可直接扫码或分享链接进入
- 可安装到桌面/主屏，接近原生体验
- 支持离线缓存“最近阅读”和“已收藏文章”
- 后续若验证成立，可平滑迁移到 React Native / Flutter 或保留 Web 主端

## 5. 内容频道设计

### 5.1 简单频道

定位：帮助用户低门槛读懂“今天发生了什么”。

推荐数据源：

- `NHK NEWS WEB EASY`

内容呈现：

- 标题
- 发布时间
- 分类
- 日文正文
- 假名标注（若源站可提供）
- 中文关键词提示
- 重点词汇卡片
- 文章难度标签：`N5-N4` / `N4-N3`

用户价值：

- 新闻更短
- 句子结构更清晰
- 适合做每日输入

### 5.2 原文频道

定位：让用户接触真实新闻文体，逐步从“能看懂”走向“能适应母语速度”。

V1 推荐上线策略：

- 原文频道直接接入 `NHK NEWS WEB` 正常新闻流
- 与 `NHK NEWS WEB EASY` 形成“同一新闻体系下的简版 + 原版”学习路径
- 抓取时优先保存标题、摘要、发布时间、分类和原文链接
- 如后续扩展其他媒体，继续沿用统一 source adapter 机制

内容呈现：

- 标题
- 来源
- 发布时间
- 原文摘要
- 难度标签：`N3-N2` / `N2+`
- “阅读原站”跳转
- 关键词拆解

## 6. 核心使用路径

### 6.1 首页

首页参考你给的 Pixso 设计稿，建议延续“内容卡片 + 顶部频道切换 + 底部轻导航”的结构。

推荐模块：

- 顶部品牌区：`今日学日语`
- 双频道切换：`简单新闻` / `原文新闻`
- 今日推荐卡
- 最新新闻列表
- 学习进度入口：连续阅读天数、今日已读篇数
- 搜索入口

### 6.2 文章详情页

详情页是核心转化页，建议优先打磨。

必须包含：

- 标题区
- 来源与发布时间
- 难度标签
- 正文分段展示
- 生词高亮
- 重点词汇卡片
- 收藏按钮
- 稍后再读
- 原文链接

增强但不复杂的学习能力：

- 点击单词显示中文释义
- 长按句子显示整句译文
- 一键切换“仅日文 / 日文+释义”

### 6.3 收藏页

用于沉淀学习资产。

- 收藏文章
- 最近阅读
- 按频道筛选

### 6.4 我的页

V1 保持轻量。

- 学习天数
- 已读篇数
- 收藏数
- 阅读偏好
- PWA 安装提示

## 7. UI/UX 方向

参考 Pixso 稿时，建议保留视觉气质，但在交互上做更偏“内容学习工具”的收敛。

### 7.1 视觉关键词

- 干净
- 柔和
- 信息密度适中
- 卡片化
- 强调“今日更新”

### 7.2 设计原则

- 首页重点不是功能入口，而是“快速进入一篇文章”
- 频道切换必须固定在首屏高位
- 卡片首行必须让用户立刻感知难度和来源
- 词汇解释不能打断阅读，应采用底部抽屉或轻浮层

### 7.3 推荐设计系统

- 8pt spacing
- 圆角卡片
- 颜色分频道：
  - 简单频道：浅绿 / 米白，表达友好和轻松
  - 原文频道：靛蓝 / 深灰，表达真实和专业
- 字体：
  - 中文：`Noto Sans SC`
  - 日文：`Noto Sans JP`

## 8. 推荐技术架构

### 8.1 前端

推荐技术栈：

- `Next.js 15` + `TypeScript`
- `App Router`
- `Tailwind CSS`
- `shadcn/ui` 仅选用少量基础组件
- `next-pwa` 或基于 Workbox 的自定义 PWA 配置
- `TanStack Query` 做客户端数据缓存

原因：

- SEO 和首屏性能更适合内容产品
- PWA 支持成熟
- 服务端拉新闻、客户端阅读体验都比较顺
- 未来扩展管理后台和多内容源成本低

### 8.2 后端

V1 建议“轻后端”模式：

- Next.js Route Handlers 作为 API 层
- 定时抓取任务单独运行
- 暂不拆独立后端服务

### 8.3 数据库

推荐：

- `PostgreSQL`
- ORM：`Prisma`

核心表：

- `sources`
- `articles`
- `article_segments`
- `article_vocabulary`
- `users`
- `user_saved_articles`
- `user_read_history`

### 8.4 抓取与定时任务

推荐：

- 部署在 `Vercel + Cron`
- 或 `Cloudflare Workers + Cron Triggers`
- 如果抓取逻辑复杂，单独放 `Node.js worker`

抓取流程：

1. 定时读取 RSS / 公开列表页
2. 拉取文章元信息
3. 去重
4. 解析正文和发布时间
5. 写入标准化表
6. 生成难度和词汇标签
7. 刷新首页缓存

## 9. 抓取与内容标准化设计

### 9.1 数据源接入层

每个来源都实现统一适配器接口：

```ts
type NewsSourceAdapter = {
  sourceKey: string;
  fetchFeedItems(): Promise<FeedItem[]>;
  fetchArticleDetail(url: string): Promise<ArticlePayload>;
  normalizeCategory(raw: string): string;
};
```

这样后续新增别的媒体或调整原文频道来源时，不会影响业务层。

### 9.2 统一文章结构

```ts
type Article = {
  id: string;
  channel: "easy" | "original";
  source: string;
  title: string;
  summary?: string;
  url: string;
  publishedAt: string;
  contentBlocks: {
    type: "paragraph" | "heading" | "list";
    text: string;
    ruby?: { base: string; reading: string }[];
  }[];
  difficultyLevel: "N5-N4" | "N4-N3" | "N3-N2" | "N2+";
  tags: string[];
};
```

### 9.3 去重规则

- 同源 URL 唯一
- 标题相似度去重
- 发布时间 + 标题哈希兜底

### 9.4 难度判定

V1 不建议上来就做复杂 AI 评级，先走规则模型：

- 句长
- 常用汉字比例
- 假名比例
- JLPT 词表覆盖率
- 来源权重

输出到 4 档难度即可。

## 10. 学习能力设计

V1 只做最有价值的三个学习增强：

### 10.1 生词卡片

点击词语展示：

- 假名
- 词性
- 中文释义
- 例句片段

### 10.2 句子辅助

- 长按句子显示中文参考译文
- 支持逐段展开，不默认全量翻译，避免用户只看中文

### 10.3 学习留存

- 收藏文章
- 已读标记
- 最近阅读恢复

## 11. API 草案

### 11.1 获取首页频道列表

`GET /api/feed?channel=easy|original&cursor=...`

返回：

```json
{
  "items": [
    {
      "id": "art_001",
      "title": "......",
      "source": "NHK",
      "difficultyLevel": "N5-N4",
      "publishedAt": "2026-04-26T08:30:00Z",
      "summary": "......"
    }
  ],
  "nextCursor": "..."
}
```

### 11.2 获取文章详情

`GET /api/articles/:id`

### 11.3 收藏文章

`POST /api/articles/:id/save`

### 11.4 阅读记录

`POST /api/articles/:id/read`

## 12. PWA 能力边界

V1 推荐支持：

- 可安装
- 首页和详情页缓存
- 最近 20 篇文章离线读取
- 图片懒加载
- 弱网降级

V1 不建议支持：

- 推送通知
- 大量离线包
- 后台音频

## 13. 合规与风险

这是项目能否上线的关键部分。

### 13.1 内容版权

- NHK 及后续新增新闻源都必须逐一核查使用条款
- 对不允许再分发全文的来源，只展示标题、摘要和跳转
- 生产环境尽量避免“爬全文后在自己 App 完整展示”

### 13.2 抓取稳定性

- 页面结构会变
- RSS 可能停更或限流
- 需要 source adapter + parser fallback + 日志告警

### 13.3 翻译与释义

- 中文释义仅做学习参考
- 不承诺官方翻译准确性

## 14. V1 页面建议

建议控制在 5 个页面以内：

- `/` 首页
- `/article/[id]` 文章详情
- `/saved` 收藏
- `/history` 最近阅读
- `/me` 我的

## 15. V1 里程碑

### 阶段 A：最小可用版

- 接入 NHK EASY
- 首页双频道 UI 完成
- 详情页完成
- 收藏和阅读记录
- PWA 安装与基础缓存

### 阶段 B：学习增强

- 词汇弹层
- 难度标签
- 句子辅助释义

### 阶段 C：稳定化

- 多来源接入
- 抓取监控
- 来源合规替换

## 16. 最终建议

从资深开发者视角，这个产品 V1 最重要的不是“功能做多”，而是把三件事做稳：

1. 内容源合法、稳定、每天真更新。
2. 用户能快速进入适合自己难度的文章。
3. 一篇文章就能形成完整学习闭环。

如果你要马上开做，我建议技术路线直接定为：

- 前端：`Next.js + TypeScript + Tailwind`
- 数据：`PostgreSQL + Prisma`
- 抓取：`Source Adapter + Cron`
- 首发内容：`NHK NEWS WEB EASY + NHK NEWS WEB`
- 终端：`PWA`

这条路线最轻、最稳，也最适合先验证产品价值。
